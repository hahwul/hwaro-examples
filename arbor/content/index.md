## About This Guide

Arbor is a comprehensive reference for Git branching strategies and team workflows. Whether you are adopting GitFlow for a release-driven project, practicing Trunk-Based Development for continuous delivery, or using GitHub Flow for a lightweight open-source workflow, this documentation provides the patterns, policies, and practical commands you need.

## What You Will Find

- **Branching models** -- side-by-side comparisons of the most widely adopted strategies, complete with diagrams and example commands.
- **Getting started guides** -- step-by-step instructions for installing Git, initializing a repository, and establishing your first branch structure.
- **Team policies** -- concrete guidelines for merge versus rebase decisions, branch naming conventions, and code review requirements.

> A good branching strategy is not about the tool. It is about how your team collaborates, delivers, and maintains confidence in every release.
