## What this track covers

The Craft Track is for engineers who care about the less-measured parts of the job: naming, review, docs, and the culture around all three. Seven sessions and three workshops.

Expected background: has reviewed more than five hundred pull requests. Has read Hoare&rsquo;s &ldquo;Hints on Programming Language Design&rdquo; at least once and does not remember it being tedious.

## Sessions

1. **Day 01 &#183; 10:00 &#183; The API review committee you can&rsquo;t fire.** How the big language and framework communities run API reviews, and what small teams can borrow.
2. **Day 01 &#183; 14:00 &#183; Docs as a first-class artefact.** Why docs should get the same review process as code, and how to do that without drowning everyone in markdown.
3. **Day 01 &#183; 16:30 &#183; Naming, 2026 edition.** Patterns, anti-patterns, and the inevitable talk about colours of functions.
4. **Day 02 &#183; 15:30 &#183; The Long Code Review.** Four engineers review the same 1,400-line diff on stage.
5. **Day 03 &#183; 10:00 &#183; Writing a changelog your users will read.** A strong defence of the semantic-version-with-prose format.
6. **Day 03 &#183; 13:30 &#183; The 20-minute pull request.** What a good PR description looks like, with before/after examples.
7. **Day 03 &#183; 15:00 &#183; Closing panel: what we changed our mind about.** Moderated round-table with four conference regulars.

## Workshops

- **Workshop 04.1** &mdash; Live API review, four hours, real-world code, no slides.
- **Workshop 04.2** &mdash; Documentation refactoring, three hours, bring your own docs.
- **Workshop 04.3** &mdash; Code review as pairing, three hours, with pre-assigned partners.

## Speakers

Tariq al-Saadi, Iris Sato, and a rotating cast of reviewers from the Django, Rust, and Kubernetes communities.
