Ohm is generally available today, after eleven months of running the
collection pipeline against beta customers' production traffic. This is the
release we designed the whole product around, so it's worth explaining what
actually shipped rather than just announcing it.

<!-- more -->

## The collection pipeline

Every pageview now runs through the rotating-salt hashing scheme described
in our [privacy page](/privacy/): IP, user-agent, and domain go in, a
one-time daily hash comes out, and the inputs are discarded before the
request finishes. This is the only identity signal Ohm ever touches, and as
of today it is the only one it will ever need — there is no roadmap item
that adds a second one.

## Where it runs

Collection, aggregation, and storage all run in Frankfurt at general
availability, with Dublin following in the next release for customers who
need a second region. No request touching visitor data crosses a border
outside the EU at any point in the pipeline.

## The dashboard

The dashboard we shipped today deliberately has no line charts, no
sparklines, and no heatmaps. Every metric is a single oversized number with
a one-word label — visitors, sessions, referrers, countries — because a
trend line invites you to stare at noise, and a number invites you to make
a decision and close the tab. If you want the trend, the CSV export (coming
in 1.3) will give you the raw series to chart yourself.

## Upgrading from beta

Beta accounts are carried over automatically with their existing project
IDs and installed script tags — no re-install required. Beta data collected
before today counts toward your plan's retention window starting from its
original collection date.
