Waypoint 1.2.0 rebuilds the on-disk image format around random access. The new
format v2 keeps a per-file table of contents next to the chunk store, so a read
of `/app/server` jumps straight to the right chunk instead of walking the layer.
This is what makes lazy pulls feel like a local disk rather than a download.

<!-- more -->

## Added

- **Image format v2.** Each image carries a TOC mapping virtual paths to chunk
  offsets. Conversion is one command and is backward compatible with pinned
  digests once re-pushed:

```bash
waypoint image convert --to v2 registry.example.com/api:1.2
```

- **Registry mirroring.** List mirrors in priority order; Waypoint probes health
  and fails over without interrupting an in-flight lazy read.

```toml
[[registry.mirror]]
host = "registry.example.com"
endpoints = ["https://mirror-a.internal", "https://mirror-b.internal"]
```

- **Metrics.** A `/metrics` endpoint reports `waypoint_cold_start_seconds`,
  `waypoint_materialized_bytes`, and `waypoint_chunk_cache_hits_total`.

## Changed

- Layer and chunk fetches share a single HTTP/2 connection per registry, so
  images with hundreds of small files no longer pay per-request connection cost.

## Removed

- The v0 image index is gone. Images still on v0 must be converted; the runtime
  prints the exact command when it encounters one.
