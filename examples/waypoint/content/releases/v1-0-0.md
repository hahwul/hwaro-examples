Waypoint 1.0.0 marks the runtime as stable. The command-line interface, the
config schema, and the on-disk image layout are now covered by a compatibility
promise: anything documented here keeps working for the life of the 1.x line.
Rootless operation and lazy pulls — the two ideas the project was built around —
are both on by default.

<!-- more -->

## Added

- **Compatibility promise.** The `waypoint` CLI and `waypoint.toml` schema are
  stable. Deprecations get a full minor-release cycle of warnings before removal.
- **userlink networking.** Each container gets its own address through a
  user-space network stack — no root, no bridge, no `NET_ADMIN`:

```bash
waypoint run --net=userlink registry.example.com/api:1.0
waypoint net status
```

## Changed

- **Lazy pulls are GA.** Every `waypoint run` mounts the image lazily unless you
  pass `--pull=eager`. The FUSE snapshotter is considered production-ready.
- **Secure by default.** Containers start under a curated seccomp profile and an
  unprivileged user namespace with no ambient capabilities. Opt into more only
  when a workload genuinely needs it:

```toml
[security]
seccomp = "default"      # default | unconfined | <path>
userns = "auto"
no_new_privileges = true
```

## Notes

Thank you to everyone who ran the 0.x betas in anger. The rough edges you filed
