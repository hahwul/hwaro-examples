Waypoint 0.9.0 is the first build we have handed to anyone outside the team. It
is an edge release: the ideas are real, the interfaces are not yet fixed. If you
are evaluating it, pin the exact digest and expect the next tag to move things
around.

<!-- more -->

## Added

- **Rootless `waypoint run`.** Pull and run standard OCI images entirely as your
  own user. There is no daemon and no setuid binary in the path:

```bash
waypoint run registry.example.com/hello:0.9
waypoint ps
```

- **Experimental lazy pulls.** With `--pull=lazy`, images mount through a FUSE
  snapshotter that fetches each layer the first time a file inside it is read.
  On a large image with a small hot set the difference to first output is
  dramatic; on a tiny image it is a wash.

```bash
waypoint run --pull=lazy registry.example.com/api:0.9
```

## Known limitations

- The snapshotter is single-threaded and fetches layers strictly in order — a
  slow layer blocks the ones behind it. Parallel fetch lands in a later build.
- Networking is host-only for now; there is no per-container address yet.
- Metrics and checkpointing do not exist. Do not run this in production.

## What is next

The road to 1.0 is about making lazy pulls fast and rootless networking real.
Parallel fetch, a prefetch heuristic, and a user-space network stack are the
next three milestones.
