Waypoint 1.2.1 is a patch release. It carries no new features and no default
changes — only fixes, one of them security-relevant. All 1.2.x users should
upgrade; pinned digests from 1.2.0 keep resolving, so the upgrade is a binary
swap.

<!-- more -->

## Security

- **CVE-2026-10118 — path traversal via TOC symlinks.** The lazy image format
  stores a table of contents that maps virtual paths to chunks. A crafted image
  could point a TOC entry at a symlink escaping the mount root, so a read of an
  in-container path could resolve to a host path. The snapshotter now canonical-
  izes every TOC target against the mount root and rejects any entry that
  escapes it. Reported by the Meridian security team; thank you.

To confirm you are patched:

```bash
waypoint version --json | jq '.snapshotter.hardened'
# true
```

## Fixed

- Cancelling a lazy read — for example when a container exits during startup —
  could leak the descriptor backing that chunk request. Descriptors are now
  released on every path out of the fetch routine.
- `waypoint ps` under-counted materialized bytes for containers created by
  `waypoint restore`; the restore path now seeds the accounting correctly.

## Upgrading

```bash
waypoint self update --channel stable
waypoint restart --all
```
