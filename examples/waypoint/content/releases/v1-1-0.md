Waypoint 1.1.0 is about latency you never see. The [1.0.0](/releases/v1-0-0/)
runtime fetched layers strictly on demand, one blocking read at a time. This
release adds a prefetch heuristic that watches the entrypoint's early reads and
pulls ahead along the same path, so by the time the process asks for the next
file it is usually already resident.

<!-- more -->

## Added

- **Prefetch heuristic.** After the first few reads, Waypoint predicts the next
  files along the directory and inode locality of the access trace and fetches
  them in the background. It is conservative by default and never evicts a chunk
  the running process still needs.
- **`waypoint stack`.** Bring up several linked containers — an API, a worker,
  a cache — from one file, each pulled lazily and started as its dependencies
  report healthy:

```yaml
# stack.yaml
services:
  api:
    image: registry.example.com/api:1.1
    needs: [cache]
  cache:
    image: registry.example.com/cache:7
```

```bash
waypoint stack up -f stack.yaml
```

## Changed

- Layer fetches now run with a bounded concurrency window rather than in strict
  order, so a slow layer no longer blocks independent ones behind it.

## Fixed

- Two containers materializing the same shared chunk at the same moment could
  deadlock on the chunk lock. The lock is now acquired in a fixed order.
