Gnomon begins with a simple objective: providing a dependency-aware static scheduler that runs reliably across large pools of heterogeneous servers. This release introduces our core execution loop, which leverages a distributed database to coordinate tasks without a single point of failure. We have engineered the coordinator to process DAGs of jobs, ensuring that downstream tasks are only queued once upstream dependencies yield a successful exit code.

<!-- more -->

A major focus of this initial version is the worker architecture. Workers establish persistent connections to the database using an efficient polling mechanism, pulling only the jobs that match their specified capabilities. To prevent task starvation, the scheduler implements a fair-share queueing algorithm that balances execution load across active node clusters. Network partitions are handled gracefully via automated lease renewals, so that aborted jobs are marked for rescheduling by healthy nodes in a timely manner. We are excited to see how teams deploy this foundation to handle their daily workloads, and we welcome any feedback regarding our configuration formats or execution metrics.
