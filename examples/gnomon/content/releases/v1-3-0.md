As workflows grow in complexity, resolving failed dependencies becomes a key operational challenge. Gnomon v1.3.0 introduces dependency-graph retries. When a parent task fails, the scheduler no longer cancels all child tasks. Instead, operators can configure partial retries that re-run only the failed node and its immediate dependents, preserving the work completed by other branches of the DAG. This dramatically reduces resource consumption on large-scale processing runs.

<!-- more -->

We have also implemented priority-based queues. Jobs can now be assigned numerical priorities, with higher-priority tasks bypassing standard queues to be executed first. This is crucial for SLA-sensitive workloads that must run immediately regardless of cluster occupancy. Finally, we have updated our cluster health monitoring dashboard. Node heartbeats now carry system diagnostics like CPU usage, disk space, and memory overhead, allowing the coordinator to avoid scheduling jobs on resource-constrained servers. These features combine to make Gnomon v1.3.0 our most stable and intelligent release yet.
