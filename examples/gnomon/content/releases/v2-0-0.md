We are proud to announce Gnomon v2.0.0, our next major release. The defining feature of this release is dynamic DAG generation. While earlier versions of Gnomon required jobs to be defined statically beforehand, v2.0.0 allows workers to generate new dependencies at runtime based on the output of previous steps. This opens up support for complex map-reduce workloads, recursive file processing, and dynamic ETL pipelines.

<!-- more -->

To support mission-critical infrastructure, we have also built multi-region failover. The coordinator can now manage workers distributed across distinct network regions. If a region becomes completely isolated due to a network partition, the coordinator automatically migrates pending workloads to alternative regions, maintaining system availability without human intervention. Finally, this release introduces a new plugin architecture. Users can extend the scheduler's functionality by writing custom processors in Rust or Go, loaded dynamically at runtime. This modularity allows Gnomon to adapt to specialized enterprise environments without bloating the core scheduler. Please review the migration guide before upgrading, as there are several breaking changes to the configuration syntax.
