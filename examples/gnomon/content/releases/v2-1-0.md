Performance and stability are the twin pillars of job scheduling. In Gnomon v2.1.0, we have focused on database polling optimization. By redesigning our query execution plans and introducing targeted index strategies, we have cut database load by forty percent. This allows a single Gnomon coordinator to manage up to ten thousand concurrent worker connections without database connection exhaustion.

<!-- more -->

Additionally, we have improved our plugin lifecycle hooks. Custom plugins can now register callback handlers that trigger before a job is dispatched, when it yields progress, and immediately upon completion. This enables deep integration with internal observability and auditing systems. We have also resolved several bugs related to dynamic DAG execution, particularly in scenarios where child tasks are generated inside heavily nested loops. These fixes ensure that runtimes remain predictable even under massive, highly parallel loads. The upgrade path is fully backward-compatible with all existing configuration schemas, requiring no manual migrations. This is a recommended upgrade for all users running v2.0.0 or higher.
