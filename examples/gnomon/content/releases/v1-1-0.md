Managing failure modes is a critical responsibility of any enterprise job scheduler. Gnomon v1.1.0 addresses this head-on by introducing fine-grained retry policies that can be configured on a per-job basis. Users can now specify exponential backoff factors, jitter ranges, and maximum retry limits directly within their job definition templates. This prevents cascading failures from overwhelming external dependencies when a service experiences a temporary hiccup.

<!-- more -->

In addition to robust retries, this release exposes a comprehensive suite of execution metrics. Nodes now track run durations, queue latency, and database connection pools in real-time, exporting these details via a lightweight Prometheus-compatible endpoint. We have also refined the internal thread worker scheduling algorithm. By optimizing how tasks are handed off between the database poller and the worker threads, we have reduced scheduler overhead by nearly twenty percent. This improvement is particularly visible in high-frequency environments where thousands of brief tasks are dispatched every minute. We recommend updating as soon as possible.
