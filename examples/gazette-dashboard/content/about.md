The Data Gazette is a broadsheet that happens to report numbers. Every reading we publish is set in ink, locked into columns, and pressed at build time. There is no live database humming behind the page; there is only the composing room, the type, and the cream stock it lands on.

We hold to the old discipline of the print desk. A figure earns its place in the table only if it is true at the moment of going to press. Headlines are written to be read, not optimized. The single red we permit ourselves marks a close, a correction, or a rule, and nothing else.

<div class="pull-quote">What is set, stays set, and renders the same on every reader's table.</div>

## The Editorial Desk

1. **One ink, one accent.** Near-black on cream, with a single editorial red. No second color buys its way onto the page.
2. **Figures align right.** Market listings are tabular, hairline-ruled, and read like a stock table. Up is solid ink; down is red.
3. **The lead carries a drop cap.** Every front-page story and every report opens with a set initial, the way a broadsheet should.

## The Plant

- **Press**: the Hwaro static engine, composed in Crystal, run on each build.
- **Type**: Playfair Display sets the nameplate and headlines; Spectral carries the body; Source Sans 3 handles the small-caps kickers.
- **Stock**: solid cream newsprint, no gradients, with an optional halftone of solid dots reserved for accent blocks.

The edition you are reading was composed and pressed in a single pass. When it is rebuilt tomorrow, the dateline will move and the figures will follow, but the discipline will not.
