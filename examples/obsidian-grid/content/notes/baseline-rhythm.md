# Baseline Rhythm

Leading is the metronome of the page. Pick a value and let everything snap to it. Headings, body, captions, code blocks, the gap above a horizontal rule. If a single element ignores the grid, the eye notices before the brain can name what changed.

The math is dull and the math is the point. Body at sixteen pixels with a leading of twenty-four. Heading at thirty-two with a leading of forty-eight. A blank line is twenty-four. A section break is forty-eight. There is no need to invent a third value.

Code blocks are the usual offender. Monospaced glyphs ship with their own ideas about line-height, and the browser cheerfully obliges them. Override the default. Force the same twenty-four pixels. The block will look slightly tight. That is correct. A loose code block is a code block that no longer belongs to the page.

When a designer asks for "a little more air" between two paragraphs, the request is almost always wrong. The air is uniform by construction. What they have noticed is something else: a heading that started its life one weight too heavy, a margin that was added in a panic, a stray descender clipping into the next line. Fix the actual cause. Do not nudge.

A grid that holds is invisible. A grid that fails is the only thing the reader sees.
