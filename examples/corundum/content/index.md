Corundum is a self-hosted identity server. It issues and verifies the tokens
that let your applications delegate sign-in without ever handling a password
directly. It speaks **OAuth&nbsp;2.1** and **OpenID Connect**, enrols hardware
security keys as a first-class factor, and runs entirely on infrastructure you
control.

This documentation reads like a specification because the subject demands it.
The flows are exact; the token formats are given in full; each ceremony is set
out step by numbered step. Read the guides in order, or consult the Security
Model when you are ready to harden a deployment.
