# About Dispatch

Dispatch is an event-driven messaging system designed for teams building distributed applications. It acts as a central event bus, decoupling producers from consumers and enabling asynchronous communication between services.

## Design Principles

- **Simplicity** -- A minimal API surface that does one thing well
- **Reliability** -- Messages are persisted to disk before acknowledgment
- **Observability** -- Built-in metrics, tracing headers, and dead letter queues
- **Portability** -- Single binary with no external dependencies

## Project Goals

Dispatch aims to provide a self-contained alternative to heavier message broker systems. It is suitable for small-to-medium deployments where operational simplicity matters more than extreme throughput.
