This line makes collectors precise. You can now pin any collector to a single
cgroup, so the agent accounts only for the workload you care about.

<!-- more -->

### Added

- **Per-collector `cgroup` filter.** Scope `blockio`, `net`, or any collector to
  one service instead of the whole host, cutting map cardinality dramatically.
- `net` collector, emitting `net.throughput` in KiB/s per interface.
- OTLP resource attributes are now populated from Kubernetes downward-API files
  when present.

### Changed

- Reduced resident memory from ~38 MiB to ~24 MiB by sharing a single BTF handle
  across all loaded programs.
- Export batches now flush on a 1 s timer rather than per-collector timers.

### Fixed

- Fixed a rare map iterator stall when a cgroup was removed mid-scrape.
- Corrected `tcp.retransmits` attribution for connections using TCP fast open.
