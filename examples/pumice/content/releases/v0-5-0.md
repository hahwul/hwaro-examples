The headline change: the sampler is no longer a fixed ratio you guess at. You
set a CPU budget and the agent solves for the rate that fits, continuously.

<!-- more -->

### Added

- **Adaptive budget controller.** `--sample adaptive` (now the default) holds
  agent CPU under a configurable `--budget`, adjusting the sample divisor every
  250 ms. See [How adaptive sampling works](@/guides/how-adaptive-sampling-works.md)
  for the loop internals.
- New `sched.runqueue` metric from the `sched` collector, exported in tasks.
- `pumice status --watch` renders a live probe and egress board in the terminal.

### Changed

- Ring buffer reads moved to a batched `epoll` path, cutting per-event wakeups
  by roughly 40% on high-syscall hosts.
- Default ring buffer size raised from 16 MiB to 64 MiB.

### Fixed

- Corrected a BTF relocation failure on kernels built without
  `CONFIG_DEBUG_INFO_BTF_MODULES`.
- `dns` collector no longer double-counts lookups served from the local cache.
