The twelve-column grid is not a design choice. It is an admission that humans write more happily when they can subdivide by two, three, four, and six without argument. A twelve-column page knows the answer to *how many columns* before the question is asked, which leaves the writer to answer the only questions that matter: *what goes first* and *what, honestly, can be cut*.

## What a grid does

1. It resolves ambiguity at the layout layer, so the writer does not have to.
2. It makes every element accountable to a position, which makes every element accountable to a reason.
3. It survives redesigns, because the structure outlives the taste.

> The grid is not a jail. The grid is an agreement with the page about where arguments go.

I have shipped designs without grids. They rarely age well. The same design, set on a twelve-column rig, ages into a style. That is the difference, and it is not a small one.
