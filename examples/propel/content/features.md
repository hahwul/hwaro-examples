Propel is built around one belief: the fastest way to process email is to never leave the keyboard. Every capability below started as a single question — what's the minimum number of keystrokes to get from "message arrives" to "message handled" — and the answer is almost always one.

That constraint shapes the whole product. There's no context menu buried three clicks deep, because a context menu means reaching for the mouse. There's no "are you sure?" dialog on archive or delete, because undo is one keystroke away and always will be. Even search doubles as a command runner, so the same reflex that finds a message can also act on it.

The eight tiles below aren't a marketing list bolted onto a slower product underneath. They're literally the shortcuts you'll use on day one, rendered the way they render inside the app itself — down to the exact keys, so there's nothing to relearn once you sign in.

New capabilities only ship once they clear that same bar. If a feature can't be reached in one or two keystrokes from wherever you're standing in the inbox, it goes back for another pass before it ever reaches a release build.
