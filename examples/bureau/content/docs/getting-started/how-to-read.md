# How to Read These Documents

**Document ID:** GS-001 | **Effective Date:** 2025-01-15 | **Owner:** Office of Governance and Compliance

---

## Article 1. Document Structure

### Section 1.1 Numbering Convention

All policy documents in Bureau follow a hierarchical numbering system:

- **Article** -- Top-level thematic division (e.g., Article 1, Article 2)
- **Section** -- Subdivision within an article (e.g., Section 1.1, Section 1.2)
- **Clause** -- Specific provision within a section (e.g., Clause 1.1.a, Clause 1.1.b)

### Section 1.2 Document Identifiers

Each document carries a unique identifier composed of a category prefix and a sequential number:

| Prefix | Category | Example |
|---|---|---|
| GS | Getting Started | GS-001 |
| POL | Policy | POL-001 |
| GOV | Governance | GOV-001 |
| STD | Standard | STD-001 |
| PRO | Procedure | PRO-001 |

## Article 2. Terminology

### Section 2.1 Normative Keywords

The following keywords carry specific meaning when used in policy documents:

- **SHALL** / **SHALL NOT** -- Absolute requirement or prohibition
- **SHOULD** / **SHOULD NOT** -- Recommended practice; deviation requires documented justification
- **MAY** -- Optional provision left to the discretion of the responsible party

### Section 2.2 Cross-References

Documents may reference other Bureau documents using their Document ID. All cross-references link to the current effective version unless otherwise stated.

## Article 3. Review Metadata

### Section 3.1 Header Block

Every document includes a metadata header containing:

1. Document ID and title
2. Effective date
3. Policy Owner
4. Approval authority
5. Review cycle information

### Section 3.2 Revision History

A revision history table appears at the end of each document, recording all amendments since initial publication.
