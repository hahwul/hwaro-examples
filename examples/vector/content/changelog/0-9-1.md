The fleet dashboard was the single most requested feature since the relay
left beta, and it is deliberately minimal: a table of devices, their last
successful sync, and how many log entries are waiting to be pulled. It reads
directly from relay cursor state, so there is nothing extra for a device to
report and nothing new for the SDK to instrument.

<!-- more -->

`--dry-run` was smaller but overdue. Teams running migrations against a
fleet wanted to see the plan before it touched a device that might be
mid-transaction on a bad connection:

```bash
$ vector schema apply ./schema.vx --dry-run
0008_add_battery_column  →  alter table devices add column battery_pct integer
0009_index_battery       →  create index on devices(battery_pct)
2 migrations pending. Nothing was applied (--dry-run).
```

The tab completion fix came from the same root cause as most completion
bugs: the shell was caching a schema snapshot from process start instead of
re-reading it after a migration ran inside the same session. It now
invalidates that cache whenever `schema apply` runs, in the REPL or from
a script.
