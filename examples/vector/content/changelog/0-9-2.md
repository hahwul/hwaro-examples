The relay crash in this release only reproduced with a specific class of
expired intermediate certificate, which is why it took three weeks in the
wild before we could reliably trigger it locally. A field device running an
old build would present a chain the relay's TLS layer had already trimmed on
its own side, and the mismatch walked off the end of a buffer instead of
failing the handshake. The fix rejects any chain the relay cannot fully
verify before touching connection state, which is also just the correct
behavior independent of the bug.

The cold start fix is unrelated but shipped in the same release because both
came out of the same fleet's incident report. A device that had been offline
for two months came back with a 6 GB write-ahead log, and `vector open`
was re-validating every record in it before serving the first read.

```bash
$ time vector open ./devices.vec
vector 0.9.1: 4.132s
vector 0.9.2: 0.071s
```

`vector open` now trusts the last checkpoint recorded in the log header and
only replays entries written after it, which is the same guarantee the
engine already gave for correctness — the old code was just re-verifying work
it did not need to redo.
