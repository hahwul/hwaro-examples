## Types of Data

The project will generate four categories of data:

1. **Observational data** obtained from partner municipalities (SCADA readings, infrastructure condition assessments, water quality measurements). These data are subject to municipal data-sharing agreements and may contain operationally sensitive information.

2. **Model outputs** from hydraulic simulations, climate downscaling, and optimization runs. These outputs will be large (estimated 15 TB over the project lifetime) and will be fully reproducible from archived code and input data.

3. **Stakeholder engagement data** including workshop transcripts, survey responses, and preference elicitation results. These data are subject to IRB protocols and participant consent agreements.

4. **Software artefacts** including source code, configuration files, container images, and documentation.

## Storage and Preservation

| Data Type | Primary Storage | Archive | Retention |
|---|---|---|---|
| Observational | MIT HPC cluster | Zenodo + MIT DSpace | 10 years post-project |
| Model outputs | Distributed across partner HPCs | Zenodo (representative subset) | 10 years post-project |
| Stakeholder data | Encrypted institutional storage | MIT DSpace (anonymised) | Per IRB protocol |
| Software | GitHub (development) | Zenodo DOI releases | Indefinite |

## Sharing and Access

- **Code**: All project code will be publicly available on GitHub from project inception, with quarterly DOI-bearing releases on Zenodo.
- **Model outputs**: Representative output datasets will be archived on Zenodo. Full outputs available upon request through the project website.
- **Observational data**: Subject to municipal data-sharing agreements. Anonymised and aggregated versions will be released where permitted.
- **Stakeholder data**: Available only in anonymised, aggregate form per IRB requirements.

## Standards and Formats

All data will adhere to FAIR principles (Findable, Accessible, Interoperable, Reusable). Specific format commitments:
- Tabular data: CSV with accompanying JSON metadata schemas
- Geospatial data: GeoTIFF, NetCDF-CF, GeoJSON
- Model configurations: TOML/YAML with schema validation
- Documentation: Markdown with automated generation from code
