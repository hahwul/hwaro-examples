## Overview

Version 3.0.0 marks the start of the 3.x line. The build pipeline was rewritten from scratch with deterministic output and a stable template engine API. Sites built on 2.x will need a one-time migration.

## Changelog

### New Features

- **Deterministic builds** -- Identical inputs now produce byte-identical output, enabling reliable cache reuse and reproducible deployments
- **Stable template API** -- Liquid-style filters and tags are now versioned; minor releases will not break templates
- **Section content trees** -- Sections inherit metadata from parent `_index.md` files for consistent navigation
- **Built-in image hashing** -- Content-addressed asset URLs for long-lived browser caching

### Improvements

- Markdown rendering switched to a streaming parser with 25% lower peak memory usage
- Front matter parsing now reports precise line numbers on syntax errors
- Default 404 template no longer leaks internal paths in error messages
- Verbose mode output is now machine-parseable JSON when stdout is not a TTY

### Bug Fixes

- Fixed taxonomy pages double-encoding URL fragments
- Resolved Windows path separators leaking into generated permalinks
- Corrected timezone handling for `date` filters on systems without `tzdata`

### Breaking Changes

| Change | Migration |
|--------|-----------|
| Removed legacy `{% raw %}` shortcode | Use `{% verbatim %}` |
| `site.config.foo` renamed to `site.foo` | Update template references |
| Default output directory is now `dist/` | Pass `--output public` to keep old behaviour |
