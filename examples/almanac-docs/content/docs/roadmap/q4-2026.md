## Milestone Overview

Q4 2026 is a stabilization milestone following the 4.0 architecture release. The focus is operational observability, plugin ecosystem maturity, and bringing community-driven feedback into the core distribution.

## Planned Features

### Observability

- **Build telemetry** -- Optional, opt-in OpenTelemetry traces for the build pipeline with span coverage of template, asset, and content phases
- **Structured diagnostics** -- Build warnings emit machine-readable diagnostic codes that can be filtered or escalated in CI
- **Health endpoints** -- The dev server exposes `/__hwaro/health` and `/__hwaro/metrics` for local profiling

### Plugin Ecosystem

| Area | Goal |
|------|------|
| Plugin registry | Public listing for community-published plugins |
| Plugin signing | Optional signature verification at install time |
| Compatibility matrix | Automated nightly tests against the top 20 published plugins |

### Documentation

- Full migration guide from 3.x to 4.x with worked examples
- Updated contribution guide aligned to the new module boundaries
- A reference site demonstrating multi-tenant configuration end-to-end

### Stability Targets

| Metric | Target |
|--------|--------|
| Build determinism | 100% reproducible across supported platforms |
| Median cold build (1k pages) | Under 4 seconds |
| Plugin API churn | Zero breaking changes within 4.x |

## Out of Scope

Features being intentionally deferred to 5.x:

- Embedded WASM runtime for plugins
- First-party CMS authoring UI
- Real-time collaborative editing
