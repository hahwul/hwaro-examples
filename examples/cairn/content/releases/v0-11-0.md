Quartz is about the two things people asked for most: expressive multi-key
sequences and a scan loop that stops burning battery when nothing is happening.

<!-- more -->

## Added

- **Leader sequences.** Press a leader key, then a short chord of keys, and Cairn
  fires a bound action. Sequences are defined in script and resolved by a trie
  the compiler builds ahead of time.

  ```lua
  local leader = require("std.leader")

  layer("base", function(k)
    k.bind("space", tap_hold(key.SPACE, layer.hold("nav")))
    leader.key(k, key.SEMICOLON, {
      ["w q"] = macro.text(":wq\n"),
      ["r r"] = action.reset_layers,
    })
  end)
  ```

- **`tap_dance`** behavior in `std.behavior`: bind up to four distinct actions to
  one, two, three, and four taps, with a configurable inter-tap term.
- **Idle scanning.** When the matrix reports no change for a configurable window,
  the scanner drops from a fixed 1 kHz poll to an interrupt-woken idle mode.

## Changed

- The event pipeline now exposes a **pre-resolve hook**, so scripts can rewrite
  or veto events before behaviors run. Useful for per-application layers driven
  by a host companion.
- Bytecode is **~18% smaller** on average after a dead-binding pass; large
  keymaps that previously overflowed flash on 128 KB boards now fit.
- `cairn build` prints a per-layer bytecode budget instead of a single total.

## Fixed

- Tap-hold no longer resolves as a hold when the key is released exactly on the
  tapping-term boundary; the comparison is now inclusive on the tap side.
- Combos spanning both halves fire reliably at high poll rates over the wired
  link.
- Fixed a panic when a layer script referenced a keycode alias defined later in
  the same file.

## Upgrade notes

Leader sequences are opt-in; existing keymaps are unaffected. Idle scanning is on
by default with a 250 ms window — set `[scan] idle_ms = 0` in `board.toml` to
restore constant polling.
