Flint is the release that made Cairn what it is. The interpreted keymap was
replaced with an ahead-of-time compiler and a compact bytecode runtime, and the
scan loop became a proper event pipeline that RGB, encoders, and displays all
plug into.

<!-- more -->

## Added

- **Ahead-of-time keymap compiler.** `cairn build` turns your `.layer` scripts
  into a flat bytecode table. There is no script interpreter on the hot path
  anymore — the runtime walks resolved events, so scan-to-report latency is
  deterministic regardless of how clever your keymap is.

  ```bash
  cairn build keymap/ --board corne
  #   base.layer   →  412 bytes
  #   nav.layer    →  188 bytes
  #   sym.layer    →  241 bytes
  #   total 841 / 16384 bytes bytecode
  ```

- **Event pipeline.** Every input — keys, encoders, and now the second thumb
  cluster — emits events through one ordered pipeline. Behaviors, RGB reactions,
  and display updates are all subscribers.
- **Encoder bindings in script.** Rotary encoders are first-class: bind rotation
  and press per layer.

  ```lua
  layer("base", function(k)
    k.encoder(1, { cw = key.VOLU, ccw = key.VOLD, press = key.MUTE })
  end)
  ```

- **Reactive RGB.** Underglow can subscribe to layer changes and keypress
  events without a companion app.

## Changed

- **Keymaps are now scripts, not arrays.** The old `keymap.c` array format is
  gone; the migration tool `cairn migrate keymap.c` emits equivalent `.layer`
  files.
- The matrix scanner is **allocation-free** after boot; all buffers are sized at
  compile time from the board definition.
- `board.toml` is now the single source of truth for pins, diode direction, and
  matrix size; these were previously split between C headers and config.

## Fixed

- Rollover is correct to 6KRO on boards that report NKRO incorrectly over the
  boot protocol.
- Layer-tap no longer strands a held layer when the base key is released first.
- Fixed drift in the peripheral-half clock that desynced RGB animations over
  long sessions.

## Upgrade notes

This is a breaking release. Run `cairn migrate` on your existing `keymap.c`
before building, and expect to hand-edit combos — the array format could not
express layer-scoped combos, so the migrator leaves them as global with a
`TODO`. Read the migration guide in the handbook first.
