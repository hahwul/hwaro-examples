## Fixed

- **CVE-2026-0142:** Path traversal vulnerability in content file loader allowed reading files outside the content directory
- Markdown renderer no longer executes embedded `<script>` tags when `safe = true`
- Static file server properly sets `X-Content-Type-Options: nosniff` header

## Security

- Content file paths are now canonicalized and validated against the project root
- Added Content-Security-Policy headers to development server
- Template `include` statements restricted to the `templates/` directory
