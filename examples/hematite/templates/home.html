Transform by transform

Full reference →

  1. ?crop=smart&fp-x=0.62&fit=cover

    Smart Crop

    Finds the subject with a saliency pass, then frames it — no fixed focal points to hand-tune.

  2. ?fm=auto&q=82&accept=avif,webp

    Format Negotiation

    Reads the Accept header and re-encodes into whatever the requesting browser decodes best.

  3. ?sharpen=auto&amount=120

    Adaptive Sharpen

    Measures the resize ratio and lays an unsharp mask sized to what was actually lost.

  4. ?colors=8&fm=png8

    Palette Reduction

    Quantizes to a scene-fit palette for icons and diagrams where photographic depth is wasted weight.

  5. ?grain=film400&dither=bayer

    Grain & Dither

    Lays ordered dither over flattened gradients so 8-bit banding never survives the pipeline.

  6. ?orient=auto&exif=strip

    Auto Orientation

    Reads the EXIF tag, bakes the rotation into pixels, then strips the tag so nothing double-rotates downstream.

One request, one pass

  • 01

    Parse the chain

    Query parameters resolve into an ordered transform list. Unknown parameters are rejected, not silently ignored — a typo in a filter chain fails loudly at request time.

  • 02

    Run the pipeline

    Each transform reads the buffer the last one left behind. Crop runs before resize; sharpen runs after, sized to whatever the resize actually threw away.

  • 03

    Cache the result

    The full chain — source, transforms, and negotiated format — hashes into one cache key, so your CDN edge stores exactly one file per unique request.
