## The Premise

Design on the web spent the last decade whispering. Soft shadows, blurred backgrounds, twelve shades of off-white, every interactive element coated in a thin film of ambiguity. The result is interfaces that feel like they were designed by someone afraid of getting in trouble.

**LOUD.CSS** is the antidote.

This blog covers neo-brutalist UI design — the contemporary web movement that borrows the raw material honesty of architectural brutalism and injects it with saturated color, thick borders, and hard-offset shadows that cast no doubt whatsoever about where a button begins and where it ends.

---

## What We Write About

The content here lands in three zones:

- **UI pattern breakdowns** — how to implement specific neo-brutalist components without making your codebase ugly in the process
- **Critique and comparison** — honest takes on when the aesthetic serves users and when it is just aesthetic
- **Tooling and workflow** — the practical side: build tools, CSS architecture, design tokens for flat color systems

No "10 best design trends" roundups. No screenshots of Dribbble shots nobody shipped. Just analysis of things that are actually built and used.

---

## The Stack

This site is built with [Hwaro](https://github.com/hahwul/hwaro), a fast static site generator. The design is entirely custom CSS — no utility framework, no component library. Every pixel of the neo-brutalist treatment you see here was typed by hand, which is the only honest way to write CSS about CSS.

The color palette is deliberately limited:

- `#f5f3e8` — off-white canvas
- `#000000` — borders, shadows, text
- `#b8a4ff` — electric lavender
- `#c6f432` — lime
- `#ff6b5e` — hot coral
- `#6ec6ff` — sky
- `#ffd23f` — sunshine

Seven colors. No gradients. The constraint is the point.
