The reference is organized the way you would read it from a cold start: the
query grammar first, then the machinery that answers it. Every page lists its
parameters as a strict mono grid, and every syntax example is punched onto tape
the way the operator console prints a parsed query before it reaches the index.

Read top to bottom for a full pass, or jump straight to the page you need from
the tree on the left. Each page carries a stability badge — `stable` grammar is
covered by the 2.x compatibility promise; `preview` may change before it lands.
