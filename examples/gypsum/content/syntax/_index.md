Six pages, one construct each, in roughly the order a new document tends to
need them. Every page follows the same shape: a short statement of the rule,
one or more worked examples pairing markdown source with Gypsum's actual
rendered output, and a note on the edge cases the CommonMark specification
resolves in a way that surprises people coming from other parsers.

This is not a full grammar — it covers the constructs a document author
reaches for daily. The complete grammar, including the block/inline parsing
model the plugin pipeline hooks into, lives in the [about](@/about.md) page
and the upstream CommonMark specification itself.
