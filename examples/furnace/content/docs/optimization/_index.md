Optimization techniques for improving application performance after profiling has identified bottlenecks. This section covers caching strategies, database query tuning, and general performance patterns.

## Optimization Rules

1. **Measure first** -- Never optimize without profiling data
2. **Target the bottleneck** -- Focus on the hottest path, not random code
3. **Validate improvements** -- Every change needs a before/after comparison
4. **Document trade-offs** -- Optimizations often trade one resource for another
