Local-first has always meant your notes live on your disk first. The fourth
folio extends that promise to a second device without asking you to trust a
server with either copy.

<!-- more -->

### Added

- Vault sync over the local network: two devices on the same Wi-Fi discover
  each other, pair with a six-digit code shown once, and merge changes
  directly, peer to peer.
- Merges use a CRDT per note body, so two people — or one person on two
  laptops — editing the same note offline reconcile automatically on
  reconnect, down to the paragraph.
- A sync log, viewable from `commonplace vault log`, records every merge
  event with a diff, so a surprising result is always explainable after
  the fact.
- Optional relay mode for syncing across networks, still end-to-end
  encrypted, still without a Commonplace account — the relay only ever
  sees ciphertext.

### Changed

- The vault format gained a `.commonplace/` metadata directory to hold
  per-note version vectors; existing vaults migrate automatically on first
  open with 0.4.0.
- Graph View now shows a small sync indicator on nodes that changed on
  another device since your last merge.

### Fixed

- A race condition where saving a note during an in-progress merge could
  drop the merge's changes has been closed; saves now queue behind an
  active merge.
- Sync pairing codes expired too aggressively on slower networks; the
  window is now 90 seconds instead of 30.

Two Hands does not do cloud backup. It does synchronization between devices
you own. Those are different problems, and folding them into one feature
tends to produce a worse version of both — that division of concerns is
staying deliberate.
