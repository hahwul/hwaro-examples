The third folio is the one most people ask for first: a picture of the
vault. Graph View draws every note as a node and every link as an edge, and
for the first time you can see the shape of what you've been writing instead
of just remembering it.

<!-- more -->

### Added

- Graph View renders the full vault as a force-directed layout; nodes sized
  by inbound link count, edges thickened by how many sentences reference the
  connection.
- Tag filters dim everything outside the selected tag rather than hiding it,
  so the shape of the surrounding vault stays visible for context.
- Clicking a node opens that note in the editor without leaving the graph —
  a split view keeps both panes live and in sync.
- A local graph mode, opened from any note, shows only that note and its
  neighbors two hops out, which turned out to be more useful day-to-day than
  the full vault view.

### Changed

- Orphan notes (introduced as a stat in 0.2.0) now render at the edge of the
  graph in a muted ring, making them easy to spot and reel back in.
- The vault index rebuild that powers the graph now runs incrementally on
  save instead of rescanning the whole vault, cutting graph refresh time
  from seconds to under 40ms on a 2,000-note vault.

### Fixed

- Graph View no longer freezes on vaults with a single densely-linked hub
  note — the layout solver now caps per-node edge influence.
- Dark backgrounds bleeding through the graph canvas on high-DPI displays
  has been corrected; the canvas now respects the vault theme.

Graph View ships with layout physics you can pause. A static graph, it turns
out, is far easier to actually read than one that's still settling.
