The fifth folio is about who gets to read the ledger. Vaults can now be
sealed at rest with a passphrase, and for the first time, the app itself can
be extended without waiting on us.

<!-- more -->

### Added

- Vault-level encryption: a passphrase derives a key via Argon2id, and every
  note file on disk is encrypted individually, so a stolen laptop reveals
  nothing without the passphrase.
- A plugin API exposing vault read/write, the link graph, and editor
  commands through a sandboxed script runtime. Plugins ship as a single
  `.cplug` file with a manifest and are installed by drag-and-drop.
- Daily notes: a configurable template opens or creates today's note on
  launch, with `{{cursor}}` marking where the caret lands.
- `commonplace vault lock` and `commonplace vault unlock` for scripting
  encryption from the command line, for people who keep vaults on removable
  media.

### Changed

- The plugin sandbox denies network access by default; a plugin must declare
  `network = true` in its manifest and the user must approve it once, per
  plugin, on install.
- Sync (0.4.0) now negotiates encryption compatibility before pairing —
  two vaults with different passphrases sync fine, since encryption is
  local, but the pairing screen now says so explicitly instead of assuming.

### Fixed

- Encrypted vaults on encrypted filesystems (which double-encrypt) were
  measurably slower to open than expected; note decryption is now batched
  instead of per-file, closing most of the gap.
- A plugin that threw during vault load could previously freeze the whole
  app; plugin execution is now wrapped so a failing plugin disables itself
  and reports an error instead.

Example plugin manifest, for anyone building against the new API:

```toml
[plugin]
name = "word-count-status"
version = "1.0.0"
entry = "index.js"
network = false

[plugin.permissions]
read_vault = true
write_vault = false
```

Locked Ledger is the release we were least willing to ship half-finished.
Encryption you can't fully trust is worse than no encryption at all, because
it changes what you do — this folio went through three external audits
before we called it stable.
