Beacon is the release the 1.x series was building toward. The OpenGL renderer that carried Semaphore from the first commit is retired, replaced by a Vulkan pipeline on Linux and Windows and a native Metal backend on macOS. This is a breaking release: read the migration notes before upgrading a machine you depend on.

<!-- more -->

## breaking

- **Config format is now TOML.** The YAML config served us badly — anchors and implicit typing caused more support threads than every renderer bug combined. Run the migrator; it converts values, comments, and key order:

```console
$ semaphore migrate-config
read   ~/.config/semaphore/semaphore.yml
wrote  ~/.config/semaphore/semaphore.toml
note   3 deprecated keys dropped: gl.vsync, gl.msaa, font.hinting
```

- **Minimum GPU baseline.** The Vulkan path requires a driver exposing Vulkan 1.2. Machines below the baseline — and headless CI boxes — fall back to `--renderer=cpu`, a software rasterizer that is correct but makes no speed promises.
- The `SEMAPHORE_GL_*` environment variables are gone, replaced by `[renderer]` keys.

## added

- The glyph atlas is a single array texture; a full screen of text renders in one draw call per pane instead of two per font.
- Present-mode selection: `renderer.vsync = "adaptive"` uses mailbox presentation where the driver offers it, cutting input latency by a frame.

## changed

- Frame pacing is driven by damage rather than a fixed clock. An idle Semaphore submits zero GPU work — measured at 0.0% GPU on a 120 Hz laptop panel.

## fixed

- Fractional display scaling no longer rounds pane widths apart from the window edge; the rightmost column of pixels belongs to a pane again.
