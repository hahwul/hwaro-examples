The heliograph release is about not doing work. The renderer now keeps a dirty bitmap per cell and re-composes only the rows that changed, so a blinking cursor in one pane no longer forces the other seven to repaint. On a four-pane layout running `htop`, `tail -f`, a compiler, and a prompt, frame cost fell 91% against [v2.2.0](/releases/v2-2-0/).

<!-- more -->

## added

- **Broadcast mode.** `ctrl+shift+b` mirrors the keyboard into every pane in the active tab. Broadcasting panes get an accent border so you always know the blast radius. Exclude a pane with `broadcast.exclude` or by focusing it and pressing `ctrl+shift+b` again.
- **Pointer-driven splits.** Pane dividers are now real hit targets: drag to resize, double-click to equalize the axis. Keyboard users keep `ctrl+shift+arrows`.
- **Font fallback hot-reload.** Installing a font no longer requires a restart; Semaphore watches fontconfig (and the macOS font registry) and rebuilds the fallback chain in place.

```toml
[panes.broadcast]
enabled = true
indicator = "border"      # border | titlebar | none
exclude = ["pane:logs"]
```

## changed

- The glyph atlas now evicts by frame age instead of insertion order. Long sessions with emoji-free workloads stay under 24 MB of VRAM.
- `semaphore --measure` prints per-pane frame timings as TSV so you can pipe them straight into `awk`.

## fixed

- Resizing a zoomed pane no longer leaks the pre-zoom layout into the restore stack.
- Sixel images taller than the scroll region clip correctly instead of smearing into the status line.

## performance

Cold start on integrated GPUs drops from 38 ms to 22 ms, most of it from moving pipeline-cache warming off the first frame. Semaphore now reliably opens before your shell finishes its own dotfiles.
