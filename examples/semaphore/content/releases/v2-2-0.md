Lantern rewrites the ligature engine from the ground up. The old pipeline re-shaped every dirty line on every frame; the new one shapes a visual cluster exactly once, caches the result by font and cluster hash, and reuses it until the text or the font changes. In a full-screen editor session the shaper now runs 40 times less often, and the difference is visible: typing latency under heavy ligature fonts drops below the frame budget for the first time.

<!-- more -->

## added

- **Per-rule OpenType toggles.** Every shaping rule can be switched independently, per font, without touching the font file:

```toml
[fonts.features]
calt = true      # contextual alternates: => -> !==
dlig = false     # discretionary ligatures stay off
ss02 = true      # slashed zero, if the font carries it
```

- **Glyph-level ligature splitting.** When the cursor or a selection lands inside a ligated cluster, Semaphore splits the glyph back into its component cells. The arrow stays an arrow until you touch it; then it behaves like the two characters it is.

## changed

- Box-drawing and block characters (U+2500–U+259F) bypass the shaper entirely and are drawn by the renderer, snapped to the pixel grid. `tmux` borders and TUI frames no longer show hairline gaps at fractional scale factors.
- The shaping cache is sized in clusters, not bytes; the default of 65,536 covers roughly forty full screens of unique text.

## fixed

- Zero-width joiner sequences in prompts no longer double the measured width of the line.
- Fallback shaping now inherits the primary font's feature toggles instead of resetting to defaults.
