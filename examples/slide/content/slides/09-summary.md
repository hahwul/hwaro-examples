# Summary

## What We Covered

1. **Why static sites** -- Performance, security, reliability, cost
2. **Project setup** -- `hwaro init` and project structure
3. **Configuration** -- Single `config.toml` for all settings
4. **Content** -- Markdown with TOML front matter
5. **Templates** -- Jinja2-compatible engine with filters
6. **Shortcodes** -- Reusable embedded components
7. **Deployment** -- Build once, host anywhere

## Key Takeaways

- Static sites are **fast, secure, and cheap** to run
- Hwaro gives you **full control** over templates and output
- Markdown-first workflow keeps content **portable**
- The ecosystem supports any hosting platform
