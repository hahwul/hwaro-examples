## Overview

The 3000 series covers database connection, query execution, and storage-related errors. These codes are primarily generated by the persistence layer and database migration tools.

---

## Error Code Reference

| Code | Severity | Description | Recommended Action |
|------|----------|-------------|--------------------|
| ERR-3000 | Critical | Database connection refused | Verify the database service is running and accessible |
| ERR-3001 | Error | Query execution timeout | Review the query performance, check for missing indices |
| ERR-3002 | Error | Deadlock detected | Retry the transaction; analyze concurrent operations |
| ERR-3010 | Error | Migration failed | Check the migration logs, manual rollback might be needed |
| ERR-3020 | Critical | Disk space exhausted | Free up disk space on the database server |
| ERR-3030 | Error | Data validation failed on insert | Verify the application data matches the database schema constraints |

---

## Detailed Descriptions

### ERR-3000: Database Connection Refused

This critical error occurs when the application cannot establish a connection to the primary database server.

**Log Example:**

```
2026-04-10 14:22:05 CRITICAL [db-manager] ERR-3000
  dsn=postgres://user@db-primary:5432/app_db
  error="connection refused"
  retry_attempt=3
```

**Diagnostic Steps:**

1. Check if the database process is running on the target server.
2. Verify that network firewalls or security groups are not blocking port 5432.
3. Validate the credentials and DSN configuration in the application's environment variables.

### ERR-3001: Query Execution Timeout

A query exceeded the configured `statement_timeout` value.

**Log Example:**

```
2026-04-10 15:40:12 ERROR [db-manager] ERR-3001
  query="SELECT * FROM large_table WHERE status = 'pending' ORDER BY created_at DESC"
  duration_ms=5000
  timeout_ms=5000
```

**Diagnostic Steps:**

1. Use `EXPLAIN ANALYZE` on the slow query to understand its execution plan.
2. Check if appropriate indices exist on the queried columns (`status` and `created_at`).
3. Ensure database statistics are up-to-date by running `ANALYZE`.
