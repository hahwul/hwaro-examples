Every lesson here runs the same loop: state a property, write a naive
implementation, watch Apatite falsify it, and read the shrink trace down to
the smallest input that still breaks the rule. Read them in order the first
time — later lessons assume the generator vocabulary the earlier ones build
up, from plain integers to command sequences. New to the project? [About
Apatite](@/about.md) covers installation and where the framework came from.
