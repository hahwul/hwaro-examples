# Vigenère Table Challenge

The Vigenère cipher resisted serious cryptanalysis for nearly three centuries. It was named after Blaise de Vigenère in the sixteenth century, though the underlying construction belongs to Giovan Battista Bellaso. The table itself is a square of twenty-six rows, each row a Caesar shift of the alphabet.

## The Challenge

The keyword is unknown but consists of a single English word, between four and seven characters. The plaintext is taken from a public domain technical manual.

```
LXFOPV EFRNHR XKLNRH AXVRGW IBSNJG TBOEZK
```

## Method

A Vigenère keystream is generated by repeating the keyword across the length of the plaintext. Each letter in the keystream selects a row of the table, which then dictates the shift applied to the corresponding plaintext letter. To break the cipher without the keyword, two classic tools apply.

The Kasiski examination looks for repeated trigrams in the ciphertext. Distances between repetitions are likely multiples of the key length. The Friedman test uses the index of coincidence to estimate the same value statistically rather than structurally.

## Notes

Once the key length is fixed, the ciphertext can be partitioned into that many columns. Each column is now a simple Caesar shift, which yields to frequency analysis. Solve each column independently, then read the keyword off the recovered shifts.

The Vigenère cipher demonstrates a recurring lesson in cryptography. A construction may appear secure under casual inspection while admitting a clean structural attack once the right invariant is identified. The repeating keyword is that invariant here.
