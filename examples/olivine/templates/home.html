Streaming ETL with backpressure that just works.

A pipeline is a living thing: a source at the root, transforms in the middle, a sink at the tip. Demand flows back up from the sink so nothing ever floods — and every sink can commit exactly once.

Three stages, one flow

  • Source

    Read from Kafka, Postgres CDC, or a generator. Every source joins the credit protocol, so it only pulls what the pipeline can absorb.

  • Transform

    Filter and map one record at a time, or gather them into event-time windows with watermarks for late data. State is checkpointed.

  • Sink

    Commit transactionally on each checkpoint barrier. Postgres, Kafka, and object stores all deliver exactly once, not just once-ish.

A whole pipeline in one file

No cluster, no coordinator. A pipeline is an ordinary Python process — wire the stages, call run(), and the runtime keeps the flow alive and in order.

Full quickstart → 
from olivine import Pipeline, sources, sinks

pipe = Pipeline("orders")

(
    pipe.source(sources.kafka(topic="orders"))
        .filter(lambda o: o.total_cents >= 5000)
        .map(lambda o: {"id": o.id, "usd": o.total_cents / 100})
        .sink(sinks.postgres(table="big_orders",
                             delivery="exactly_once"))
)

pipe.run()
