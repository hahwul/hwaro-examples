Brutalist web design strips away the unnecessary. No gradients, no animations, no parallax scrolling. What remains is structure and content.

## What Is Brutalism?

In architecture, brutalism means raw concrete and exposed structure. On the web, it means:

1. **Raw HTML** — semantic markup, no framework overhead
2. **Minimal CSS** — enough to be readable, nothing more
3. **No JavaScript** — if the page can exist without it, it should
4. **Default elements** — browser defaults are underrated

## The Case for Less

Every line of CSS is a maintenance burden. Every script is a potential failure point. Every dependency is a security surface.

> The best code is no code at all.

A 200-line stylesheet is easy to understand, easy to maintain, and easy to customize. You can read it top to bottom in five minutes. Try that with a modern CSS framework.
