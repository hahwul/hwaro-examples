Milepost learns to talk to the outside world. Plugins can register webhooks and
emit outbound events, the server exposes a metrics endpoint for your monitoring
stack, and the world generator got a meaningful speed-up through a reworked noise
cache.

<!-- more -->

## Added

- **Outbound webhooks.** Plugins declare webhook targets in their manifest and fire
  them with a typed payload. Delivery is retried with exponential backoff and
  surfaced in `milepost status`.

  ```typescript
  export default definePlugin({
    name: "join-bell",
    on: {
      "player.join": (ctx, e) => {
        ctx.webhook("ops-channel", {
          text: `${e.player.name} rolled into ${e.world}`,
        });
      },
    },
  });
  ```

- **Prometheus-style metrics** at `/metrics`, off by default, bound to a separate
  interface so you never expose it to players by accident.

  ```toml
  # milepost.toml
  [metrics]
  enabled = true
  bind = "127.0.0.1:9750"
  ```

- **`player.move` throttling controls** so movement-heavy plugins can opt into a
  coarser sample rate instead of an event per centimeter.

## Changed

- Worldgen noise is cached per-octave and reused across adjacent chunks; median
  generation time for a fresh 16×16 chunk dropped from 3.1 ms to 1.7 ms.
- The default view distance moved from 10 to 12 chunks now that generation is
  cheaper; override it with `[world] view_distance` if your bandwidth is tight.

## Fixed

- Fixed water and lava flow occasionally stopping at chunk borders until a
  neighboring block update nudged them.
- Entity AI pathfinding no longer pins a core to 100% when a mob is boxed into a
  1×1 space with no valid path.
- Fixed a config parse error when a world name contained a hyphen.

## Upgrade notes

Backward compatible. The metrics endpoint and webhooks are opt-in. If you pin view
distance for bandwidth reasons, set it explicitly before upgrading — the default
changed.
