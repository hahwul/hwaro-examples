The first feature release on the API v2 foundation. Milepost gets a built-in
marketplace client so operators can install plugins without wrangling files by
hand, cryptographic signing so you know what you are running, and a transactional
block buffer that makes large edits atomic.

<!-- more -->

## Added

- **Marketplace client.** `milepost plugin add <name>` fetches a plugin from a
  configured registry, verifies its signature, and installs it into the sandbox.
  Registries are just static indexes, so anyone can host one.

  ```bash
  $ milepost plugin add ore-census
  → resolving ore-census from registry "central"
  → signature ok (key: 9F2A…C71D, publisher: quarry-labs)
  ✓ installed ore-census@1.0.0
  ```

- **Signed plugins.** Every plugin ships an Ed25519 signature over its bundle. The
  loader refuses unsigned plugins unless `[plugins] allow_unsigned = true` is set for
  local development.
- **Transactional block buffer.** `world.edit()` opens a transaction; changes are
  staged and either committed atomically or rolled back, so a crashed bulk edit can
  never leave the world half-modified.

  ```typescript
  const tx = ctx.world.edit();
  for (const pos of sphere(center, 8)) tx.set(pos, "core:glass");
  await tx.commit();   // one flush, all-or-nothing
  ```

## Changed

- Plugin sandboxes now get a per-plugin memory ceiling (`64 MiB` default) enforced
  by the WebAssembly runtime; exceeding it unloads the plugin rather than the server.
- Console tab-completion learned plugin command names and player handles.

## Fixed

- Fixed inventory desync when two plugins mutated the same container within one tick.
- Chunk unload no longer cancels a scheduled task that was mid-execution.
- The `/seed` command reports the actual generation seed after a datapack override,
  not the configured one.

## Upgrade notes

If you develop plugins locally, set `[plugins] allow_unsigned = true` in a dev
profile — production refuses unsigned bundles. Bulk-edit plugins should adopt
`world.edit()`; the old per-block `setBlock` still works but is no longer atomic
across a batch.
