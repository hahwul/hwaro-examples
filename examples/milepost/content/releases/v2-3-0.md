Iteration speed for plugin authors takes a big step. Plugins hot-reload without a
server restart, the scheduler applies backpressure instead of piling up tasks, and
each world gets its own tick budget so a busy creative build can't starve survival.

<!-- more -->

## Added

- **Hot reload.** `milepost plugin reload <name>` swaps a plugin's module in place,
  replaying its declared subscriptions against the live event bus. State declared
  through `ctx.state` survives the swap; everything else is torn down cleanly.

  ```bash
  $ milepost plugin reload ore-census
  → unsubscribed 3 handlers, drained scheduler, loaded ore-census@1.4.0
  ✓ reloaded in 84ms (state preserved: 2 keys)
  ```

- **Scheduler backpressure.** `ctx.schedule.every("5s", fn)` now skips a run if the
  previous one is still in flight and records the overrun, rather than queuing a
  backlog that never drains.
- **Per-world tick budgets** in config, so heavy worldgen on one shard cannot blow
  the global 50 ms frame.

  ```toml
  # milepost.toml
  [[worlds]]
  name = "survival"
  tick_budget_ms = 40

  [[worlds]]
  name = "creative"
  tick_budget_ms = 12   # capped: builders don't need physics headroom
  ```

## Changed

- Plugin manifests moved from `plugin.json` to `plugin.toml` for consistency with
  server config. The loader reads both this release; JSON manifests warn.
- Event dispatch is now **depth-limited** — a handler that edits a block which
  triggers the same event no longer recurses past a configurable depth of 8.
- `milepost status` gained a `--json` flag for scraping into external dashboards.

## Fixed

- Fixed a memory leak in the lighting engine where relit chunks near a world border
  retained their previous sky-light array.
- Piston-like moving blocks no longer desync when two of them push into the same
  cell on the same tick; the resolver now orders by insertion, deterministically.
- Console color codes stopped bleeding into log files written with `--no-color`.

## Upgrade notes

Backward compatible. Convert manifests with `milepost plugin migrate` when
convenient. If you relied on unbounded recursive events (rare, and usually a bug),
raise `[events] max_depth` explicitly — the default of 8 is new.
