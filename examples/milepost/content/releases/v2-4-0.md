The biggest quality-of-life release since the API v2 rewrite. Plugins can now
stream chunks asynchronously without stalling the tick, permissions are a typed
first-class surface instead of string lookups, and region protection was rebuilt
around an interval tree that stops chewing CPU on worlds with thousands of claims.

<!-- more -->

## Added

- **Async chunk access from plugins.** `world.chunks.load()` returns a promise
  that resolves off the main tick, so terrain scanners and map renderers no longer
  freeze the server while they page regions off disk.

  ```typescript
  import { definePlugin } from "@milepost/api";

  export default definePlugin({
    name: "ore-census",
    commands: {
      census: async (ctx) => {
        const chunk = await ctx.world.chunks.load(ctx.player.chunkPos);
        const diamonds = chunk.count("core:diamond_ore");
        ctx.reply(`This chunk holds ${diamonds} diamond ore.`);
      },
    },
  });
  ```

- **Typed permissions API.** `ctx.player.can("region.build", { at: pos })` replaces
  ad-hoc string checks. Permission nodes are declared in the plugin manifest and
  validated at load, so a typo fails the build instead of silently denying players.
- **`milepost claims` CLI** — inspect, transfer, and prune region claims from the
  console without a plugin. Supports `--dry-run` and `--older-than 30d`.

## Changed

- **Region protection** now uses an interval tree keyed on chunk columns. A world
  with 8,000 claims dropped its per-block permission check from a median 340 µs to
  11 µs. Large factions servers should feel this immediately.
- The block buffer flush is **coalesced per tick** — bulk edits from mass-editing
  plugins commit in one pass rather than one transaction per block.

  ```toml
  # milepost.toml
  [world.buffer]
  coalesce = true       # merge plugin edits within a tick (new default)
  max_pending = 262144  # blocks queued before a forced flush
  ```

## Fixed

- Fixed a race where a plugin unloaded mid-tick could leave a dangling event
  subscription that leaked a chunk reference until restart.
- `/tp` across dimension boundaries no longer duplicates the player entity when the
  destination chunk was still generating.
- The RCON handler stopped truncating responses longer than 4 KiB.

## Upgrade notes

Drop-in for `2.x`. Plugins using the old string permission checks keep working
through a compatibility shim that logs a deprecation once per node; migrate to the
typed API before `3.0`. Run `milepost claims --dry-run` after upgrading to confirm
the interval tree rebuilt your claim index correctly.
