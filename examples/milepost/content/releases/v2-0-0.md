The interchange. Milepost 2.0 replaces the old in-process plugin model with a
sandboxed runtime, a typed event bus, and a scheduler the server controls. This is
a breaking release — v1 plugins do not load — but it is the foundation everything
since is built on.

<!-- more -->

## Added

- **Sandboxed plugin runtime.** Plugins are WebAssembly or TypeScript modules run
  in an isolate with no ambient filesystem or network access. Capabilities are
  granted explicitly through the manifest, so a plugin can only touch what you let it.

  ```toml
  # plugin.toml
  name = "ore-census"
  version = "1.0.0"
  entry = "dist/main.js"

  [capabilities]
  world = ["read"]         # read blocks, not write
  commands = ["census"]    # may register /census
  network = false          # no outbound sockets
  ```

- **Typed event bus.** Every event carries a typed payload generated from the
  server schema, so `player.join`, `block.break`, and `chunk.load` are checked at
  build time instead of discovered at runtime.

  ```typescript
  import { definePlugin } from "@milepost/api";

  export default definePlugin({
    name: "welcome",
    on: {
      "player.join": (ctx, e) => ctx.broadcast(`${e.player.name} joined`),
    },
  });
  ```

- **Runtime-owned scheduler.** The server, not the plugin, decides when scheduled
  work runs, and enforces a tick budget so no plugin can stall the world.

## Changed

- The tick loop was rewritten around a fixed 20 Hz step with a decoupled network
  thread; block updates and entity ticks no longer share a queue.
- Configuration consolidated into a single `milepost.toml`; the old
  `server.properties` file is read once and migrated with a warning.

## Removed

- The v1 in-process plugin loader and its global `Server` object are gone. There is
  no compatibility shim — v1 plugins must be rewritten against the v2 API.

## Upgrade notes

**This release breaks v1 plugins.** Budget time to port them: most map to the new
event names directly, but anything that reached into server internals needs a
capability grant instead. Back up your worlds before upgrading. Run
`milepost migrate config` to convert `server.properties`, then start with
`--dry-run` to see the plugin load report before going live.
